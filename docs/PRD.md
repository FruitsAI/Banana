# 产品需求文档 (PRD)

## 1. 产品概述
**产品目标：** 打造一款轻量、极具未来感的桌面 AI 客户端助理应用（类似 Cherry Studio 的早期形态）。
**核心卖点：** 
1. **纯净强大的对话体验：** 主打 AI 对话，同时深度集成 MCP (Model Context Protocol) 扩展能力。
2. **极简超凡的美学设计：** 采用 macOS 26 “液态玻璃” (Liquid Glass) 设计语言，提供跨平台的绝佳视觉反馈。

## 2. 核心功能及需求

### 2.1 对话核心引擎
- **多会话管理：** 侧边栏列表展示历史对话（具备选中的立体高亮效果），支持新建、重命名、删除。
- **消息交互界面：** 对话气泡、流式输出渲染、消息发送输入框及其配置按钮（支持回车发送，组合键换行）。
- **MCP 调用支持：** 允许 AI 模型在使用过程中触发用户配置的 MCP 服务器接口，执行本地或网络操作。对话区需针对 MCP 返回结果呈现专属的结构化 UI（如状态卡片）。

### 2.2 配置中心 (Settings)
- **模型提供商配置 (Provider config)：**
  - 支持配置多家主流大模型 API 服务商（如 OpenAI, Anthropic, Gemini, 乃至本地的 Ollama, LM Studio 等）。
  - 用户可填入 API Key, Base URL，并选择当前使用的默认模型。
- **MCP 服务配置 (MCP config)：**
  - 可手动添加/启用 MCP Server。
  - 支持填写启动命令（比如 `npx ...` 或本地二进制路径）。

### 2.3 跨平台与体验
- **跨平台支持：** macOS, Windows, Linux。
- **原生沉浸：** 适配操作系统的窗口控制逻辑（如 macOS 分割红绿黄按钮）。
- **多语言响应机制：** 响应式布局自适应调整界面边栏。

## 3. 设计语言与界面要求 (液态玻璃)

全应用必须保持极高的视觉水准：

1. **毛玻璃层级架构 (Glassmorphism)：**
   - 窗口底层通过半透明磨砂材质渲染桌面壁纸。
   - 界面分层：底座基础色 (`glass-base-bg`) -> 升起元素 (`glass-raised-bg`) -> 上层面板 (`glass-overlay-bg`)，层层叠加配合内阴影 (`inset-hi`, `inset-lo`)。
   
2. **渐变与质感呈现：**
   - 引入动态光效（Aurora drift, Grain drift）。
   - 按压/悬停均需伴随物理微动效（微弱的吸附形变、透光率变化）。

3. **布局骨架：**
   - **顶栏 (Titlebar)：** 容纳 macOS 交通灯，融合全局呼出的命令浮窗/搜索枢纽 (`Cmd+K`)。
   - **左一区 (Rail)：** 贴边的主导航图标，极简剔透。
   - **左二区 (Threads)：** 会话列表页面。
   - **主操作区 (Stage)：** 结合浮动感输入框 (Composer) 的主要对话显示区域。

## 4. 阶段规划
- **Phase 1:** 完成基础技术栈搭建、静态玻璃质感还原、及基础配置结构梳理。
- **Phase 2:** 接入本地模型服务及 OpenAI 规格接口，跑通对话生命周期（含流式输出）。
- **Phase 3:** 跑通 MCP Server 唤起逻辑，支持调用 MCP 工具处理上下文。
